# GitHub Actions Cron 时间参考

## 当前设置
```yaml
cron: '0 0 * * *'  # UTC 00:00 = 北京时间 08:00
```

## 推荐的替代方案

### 选项 1: 错开高峰（推荐）
```yaml
# 提前到早上 7:30 (UTC 23:30 前一天)
cron: '30 23 * * *'  # UTC 23:30 = 北京时间 07:30
```

### 选项 2: 多次执行作为备份
```yaml
schedule:
  - cron: '0 0 * * *'   # 北京时间 08:00
  - cron: '30 0 * * *'  # 北京时间 08:30（备份）
```

### 选项 3: 午夜执行
```yaml
cron: '0 16 * * *'  # UTC 16:00 = 北京时间 00:00（午夜）
```

## Cron 语法说明
```
* * * * *
│ │ │ │ │
│ │ │ │ └─ 星期几 (0-6, 0=周日)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日期 (1-31)
│ └─────── 小时 (0-23, UTC 时间)
└───────── 分钟 (0-59)
```

## 时区转换
- UTC 00:00 = 北京时间 08:00
- UTC 23:00 = 北京时间 07:00
- UTC 16:00 = 北京时间 00:00（午夜）
- UTC 12:00 = 北京时间 20:00

## GitHub Actions Cron 的已知限制

1. **延迟问题**：
   - 可能延迟 3-60 分钟
   - 高峰时段延迟更严重
   - 无法保证精确时间

2. **仓库活跃度**：
   - 60 天无活动会自动禁用
   - 需要手动重新启用

3. **最小间隔**：
   - 最快每 5 分钟执行一次
   - 不建议过于频繁

## 建议

对于每日新闻简报这种应用：
- ✅ 使用 cron 定时触发
- ✅ 同时保留 workflow_dispatch 手动触发
- ✅ 如果时效性很重要，可以设置多个时间点
- ✅ 在 Actions 页面启用通知，及时发现问题
